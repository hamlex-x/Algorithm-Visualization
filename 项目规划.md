# 命名规范
## 函数
1. 动词明确 — create/render/execute/undo/generate/calc/bind，一个动词只做一类事
2. 对象紧跟 — 动词后马上说清楚操作什么：Graph、BfsStep、Queue、CircleLayout
3. 算法名在对象后 — executeBfsStep 不是 BfsStepExecute
4. SVG/DOM 细节不出现在函数名里 — renderQueue 不给 createQueueInSVG，渲染手段是实现细节
## 常量
- [模块]_[用途]_[分类]
## 文件
- shared/：`[职责].ts`（如 types.ts、config.ts）
- builders/：`[结构]-[表示]-builder.ts`（如 graph-matrix-builder.ts、tree-array-builder.ts）
- algos/：`[结构]-[表示]-[算法]-algo.ts`（如 graph-matrix-bfs-algo.ts、sort-quick-algo.ts）
- renderers/：`[结构]-[算法]-renderer.ts`（如 graph-bfs-renderer.ts，不区分表示方法）
- pages/：`[结构]-[算法]-page.ts`（如 graph-bfs-page.ts，不区分表示方法）
- 全部小写短横线连接，不使用下划线或驼峰
- 单一表示时省略表示层（如 linked-list-builder.ts 不加 single）
